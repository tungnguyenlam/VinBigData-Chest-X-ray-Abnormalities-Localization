# Ensemble Stacking

This directory contains the logic for fusing the predictions of multiple individually trained object detection models (e.g., YOLO, Faster R-CNN, DETR) into a single, combined set of predictions. 

By default, we use **Weighted Box Fusion (WBF)**, which merges overlapping boxes from different models based on their confidence scores, rather than just discarding the lower-scoring boxes like NMS does. 

## Why do we tune WBF on the validation set, not the training set?
The ensemble weights determine how much "trust" to place in each model. 
If we tuned these weights on the **training set**, models that overfit their training data would appear artificially confident and accurate, causing the ensemble to heavily weight them. 

By instead tuning the weights on the **validation (hold-out) set**:
1. We evaluate how the models generalize to unseen data.
2. We assign higher weights to the models that perform better on unseen data.
3. We avoid "leakage" where the ensemble memorizes the training data distribution.

## Workflow Steps

To properly use this directory, you need predictions generated by each model first.

### Step 1: Generate Validation Predictions (for tuning)
Run each model's test script on the `val` split. This will produce a `.jsonl` file containing the predictions for the validation set.

```bash
# YOLO
python scripts/yolo/test.py --weights <path_to_yolo_best.pt> --split val

# Faster R-CNN
python scripts/faster_rcnn/test.py --weights <path_to_frcnn_best.pt> --split val
```
*(Produces: `outputs/yolo/predictions_val.jsonl` and `outputs/faster_rcnn/predictions_val.jsonl`)*

### Step 2: Tune Ensemble Weights (`tune_weights.py`)
Now that you have validation predictions for both models, you need to find the optimal combination weight for them. 

Run `tune_weights.py` passes the JSONL files you just generated. It will output a file called `tuned_weights.json` in the `outputs/ensemble/val/` directory.

```bash
python scripts/stacking/tune_weights.py \
    --yolo-pred outputs/yolo/yolo/predictions_val.jsonl \
    --frcnn-pred outputs/faster_rcnn/faster_rcnn/predictions_val.jsonl \
    --grid-search \
    --grid-step 0.1 \
    --grid-max 1.5
```
*(This command sweeps through different weights to find the highest fused mAP, and saves it).*

### Step 3: Generate Test Predictions 
You now know the optimal weights, but you need the raw model predictions on the **test** set so you can fuse them. Generate them just like you did in Step 1, but for the test split.

```bash
# YOLO
python scripts/yolo/test.py --weights <path_to_yolo_best.pt> --split test

# Faster R-CNN
python scripts/faster_rcnn/test.py --weights <path_to_frcnn_best.pt> --split test
```

### Step 4: Apply the Tuned Weights to the Test Set (`test.py`)
Finally, you run the ensemble stacking on your newly generated test set predictions, using the weights you calculated in step 2. 

The `test.py` script automatically reads the saved `tuned_weights.json` and applies it to the new `test` split predictions.

```bash
python scripts/stacking/test.py \
    --split test \
    --yolo-pred outputs/yolo/yolo/predictions_test.jsonl \
    --frcnn-pred outputs/faster_rcnn/faster_rcnn/predictions_test.jsonl \
    --tuned-weights outputs/ensemble/val/tuned_weights.json
```

This will run WBF on the test set predictions using the tuned weights, calculate the final mAP and FROC score, and save the fused boxes to `outputs/ensemble/test/fused_predictions_wbf.jsonl`.
